## What Are Core Web Vitals?

Core Web Vitals are performance metrics established by Google to assess:

- How fast a website loads  
- How responsive it is  
- How visually stable it remains  

They focus on real user experience rather than just technical performance.

Key questions:
- Does it load quickly?
- Does it respond instantly?
- Does it stay visually stable?

---

## Why Google Launched Core Web Vitals

Google introduced these metrics to shift focus from technical speed scores to actual user experience.

They prioritize:
- Smooth loading  
- Instant interaction  
- Stable visuals  

---

## Evolution of Core Web Vitals in 2026

- FID has been replaced by **INP (Interaction to Next Paint)**  
- Real user (field) data is now more important  
- Performance expectations are higher due to faster devices and networks  

---

## The Three Core Web Vitals Metrics

### 1. Largest Contentful Paint (LCP)

Measures how long it takes to load the main content (e.g., banner, headline).

**Thresholds:**
- Good: Under 2.5 seconds  
- Needs Improvement: 2.5 – 4 seconds  
- Poor: Over 4 seconds  

---

### 2. Interaction to Next Paint (INP)

Measures overall responsiveness of user interactions.

**Why it replaced FID:**
- FID only measured the first interaction  
- INP evaluates all interactions during a session  

---

### 3. Cumulative Layout Shift (CLS)

Measures visual stability (unexpected layout shifts).

**Scoring:**
- Good: ≤ 0.1  
- Needs Improvement: 0.1 – 0.25  
- Poor: > 0.25  

---

## Core Web Vitals Benchmarks

| Metric | Good | Needs Improvement | Poor |
|--------|------|------------------|------|
| LCP    | ≤ 2.5s | 2.5s – 4s | > 4s |
| INP    | ≤ 200ms | 200ms – 500ms | > 500ms |
| CLS    | ≤ 0.1 | 0.1 – 0.25 | > 0.25 |

## Common Causes of Poor Core Web Vitals

- Unoptimized images and media  
- Excessive JavaScript  
- Slow server response  
- Poor hosting  
- Layout shifts from ads/content  
- Improper caching

## How to Improve Core Web Vitals

### Fixing LCP Issues
- Optimize images  
- Use faster hosting  
- Implement CDNs  
- Reduce render-blocking CSS/JS  

### Improving INP
- Optimize JavaScript  
- Break long tasks into smaller chunks  
- Reduce third-party scripts  

### Reducing CLS
- Define image/video dimensions  
- Avoid inserting content above existing elements  
- Maintain stable ad placements
